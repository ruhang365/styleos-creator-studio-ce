# Permissions

MVP permissions are intentionally simple.

## Creator Owns

- services
- fan cases
- reports
- feedback
- creator notes

Creator can:

- create and edit services
- review intake
- edit tags
- accept or reject rule matches
- edit reports
- deliver reports
- review feedback
- propose candidate knowledge from their own cases

## Maintainer Can Review

- candidate knowledge
- public rule candidates
- schema suggestions

Maintainer cannot use private creator data outside the authorized review scope.

## Fan / Client Can

- submit intake
- view delivered report
- submit feedback

Fan / Client cannot access creator workspace data or candidate knowledge queue.

## Future Roles

Future Cloud may add team roles, admin roles, billing roles, and expert reviewer roles.
