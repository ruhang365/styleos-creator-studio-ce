# Open Core Strategy

## Layers

| Layer | License / model | Purpose |
| --- | --- | --- |
| `styleos-protocol` | Apache-2.0 | Open standard, schemas, starter rules, examples, and community trust. |
| `styleos-creator-studio-ce` | AGPL-3.0 | Open-source community edition product and self-hostable workflow reference. |
| StyleOS Cloud | Hosted commercial | Future managed service for small-B creators who do not want to self-host. |
| StyleOS Pro | Closed commercial | Future advanced knowledge base, expert model library, and verified case evidence. |
| StyleOS Certified | Future certification | Potential certification layer for creators, experts, and institutions. |

## Boundary

Open-source licensing does not grant rights to:

- ruhang365 trademarks
- StyleOS trademarks
- StyleOS Cloud commercial service
- StyleOS Pro knowledge library
- expert model library
- certified partner network
- private verified case database

## Why Open Core

The protocol and CE product make the standard inspectable and trustworthy. Cloud and Pro can fund reliability, hosted operations, advanced review, and expert knowledge without closing the base workflow.

## Current Commercial Stance

Alpha is not a paid launch. The current focus is workflow validation and creator feedback.
