# Case Management Workflow

Case management helps creators track each service request from intake to feedback.

## Case Objects

A case may include:

- service reference
- intake answers
- status
- selected tags
- matched rules
- report draft
- execution card
- feedback
- candidate knowledge extraction status

## Case States

See [State Machine](../specs/state-machine.md).

## Privacy Boundary

FanCase data is private creator workspace data. Only anonymized and reviewed mappings may move into Candidate Knowledge Queue.
