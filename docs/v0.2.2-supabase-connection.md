# v0.2.2 Supabase Connection & Shareable Workflow

## What Was Added

- Optional Supabase Mode.
- Storage adapter abstraction.
- Supabase browser auth helpers.
- Server-side Supabase helper for route handlers.
- Shareable intake route.
- Shareable report route.
- Cloud feedback route.
- Setup page.
- Storage mode documentation.
- Candidate knowledge cloud support.

## What Works

- Local Mode remains the default.
- Supabase Mode can use `auth.users` for creator login.
- Creator-owned services, cases, reports, feedback, and candidate knowledge can use the `styleos` schema.
- Fans can submit intake, open reports, and submit feedback through token routes without logging in.

## What Remains Local

- Local Mode seed data and reset workflow.
- Local creator profile editing.
- Browser-only demos when Supabase is not configured.

## What Requires Supabase Configuration

- Magic link login.
- Cross-device intake.
- Cross-device report sharing.
- Cross-device feedback.
- Cloud candidate knowledge queue.

## Current Limitations

- Hairstyle workflow only.
- No payments.
- No AI API.
- No image upload.
- No raw photo storage.
- No workspace or team model.
- Custom `styleos` schema may need to be exposed in Supabase API settings.
