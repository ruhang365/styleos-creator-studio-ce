# Fan Intake Workflow

Fan intake collects only the information needed for the current service.

## Steps

1. Fan opens intake link.
2. Fan reviews service scope and privacy notice.
3. Fan submits scenario, goals, constraints, maintenance willingness, and optional notes.
4. Creator receives a FanCase.
5. Creator reviews inputs before tagging.

## Photo Boundary

If photo support is added in the future, photos should be used only for the current service by default.

Photos do not enter the public repository, public protocol library, or StyleOS Pro library by default.

## Data Minimization

The fan side should not be asked for identity data unrelated to the service.
