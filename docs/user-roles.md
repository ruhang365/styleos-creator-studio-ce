# User Roles

## Creator

- permissions: create services, receive fan cases, edit tags, match rules, generate reports, collect feedback.
- boundary: owns their workspace data; cannot publish fan data without authorization.

## Fan / Client

- permissions: submit intake, view delivered report, submit feedback.
- boundary: only provides data needed for the current service.

## Maintainer

- permissions: review candidate knowledge, public rule candidates, schema suggestions, and repository contributions.
- boundary: must not access private creator data unless explicitly authorized.

## Contributor

- permissions: submit documentation, synthetic examples, schema alignment, workflow improvements.
- boundary: cannot submit personal data, API keys, unauthorized cases, or Pro content.

## Reviewer

- permissions: review workflow clarity, privacy safety, and contribution quality.
- boundary: cannot upgrade evidence level without review basis.

## Future Expert Reviewer

- permissions: review advanced rules and candidate knowledge.
- boundary: future role; not part of current CE implementation.

## Future Cloud Admin

- permissions: operate hosted workspaces, billing, support, and governance.
- boundary: future Cloud role; not part of current CE implementation.
