# Daily Alpha Checklist

Use this checklist during the small-B Alpha period.

## Health

- [ ] Check hosted Alpha URL.
- [ ] Check `/api/health`.
- [ ] Confirm storage mode is Supabase.
- [ ] Confirm Alpha mode is enabled.

## Auth

- [ ] Open `/login`.
- [ ] Confirm magic link entry is visible.
- [ ] Confirm no login errors were reported.

## Data

- [ ] Check `styleos` table counts.
- [ ] Check for unclean synthetic markers.
- [ ] Check suspicious rows.
- [ ] Confirm no real photos or sensitive personal data were reported.

## Product

- [ ] Review tester feedback.
- [ ] Review Candidate Knowledge queue.
- [ ] Review consent-related issues.
- [ ] Review report and Barber Brief issues.

## Operations

- [ ] Review Vercel deployment status.
- [ ] Review runtime errors.
- [ ] Record P0/P1/P2 issues.
- [ ] Decide whether to continue, pause, or fix before the next tester.
